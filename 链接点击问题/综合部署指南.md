# Next.js 15 静态导出链接问题综合部署指南

## 执行摘要

本指南提供了解决 Next.js 15 静态导出站点中链接点击问题的完整解决方案。当使用 `output: 'export'` 配置时，Next.js 的客户端导航可能会因为路由预取失败或资源加载问题而无法正常工作。

### 核心问题
- 静态导出的站点中，Link 组件点击后无响应
- 页面需要手动刷新才能导航
- 控制台可能出现 404 错误或资源加载失败

### 可用解决方案概览
1. **方案A：强制使用浏览器导航**（最简单，推荐用于快速修复）
2. **方案B：自定义客户端导航钩子**（平衡性能与兼容性）
3. **方案C：优化构建配置**（适合需要保留 Next.js 功能的项目）
4. **方案D：服务器端配置**（适合有服务器控制权的部署）

## 方案A：强制使用浏览器导航

### 适用场景
- 需要立即修复问题
- 对页面加载性能要求不高
- 静态站点，无需客户端状态管理

### 实施步骤

1. **创建自定义 Link 组件**

```tsx
// components/CustomLink.tsx
'use client'

import { useCallback } from 'react'

interface CustomLinkProps {
  href: string
  children: React.ReactNode
  className?: string
  target?: string
  rel?: string
}

export default function CustomLink({ 
  href, 
  children, 
  className,
  target,
  rel
}: CustomLinkProps) {
  const handleClick = useCallback((e: React.MouseEvent<HTMLAnchorElement>) => {
    // 对于外部链接，使用默认行为
    if (href.startsWith('http') || href.startsWith('//')) {
      return
    }
    
    // 对于内部链接，阻止默认行为并使用 window.location
    e.preventDefault()
    window.location.href = href
  }, [href])

  return (
    <a 
      href={href} 
      onClick={handleClick}
      className={className}
      target={target}
      rel={rel}
    >
      {children}
    </a>
  )
}
```

2. **全局替换 Link 组件**


### 优点
- 实施简单，立即生效
- 完全避免客户端路由问题
- 兼容所有静态托管服务

### 缺点
- 失去客户端导航的性能优势
- 每次导航都会重新加载页面
- 无法保持客户端状态

## 方案B：自定义客户端导航钩子

### 适用场景
- 需要检测导航失败并提供降级方案
- 希望在某些情况下保留客户端导航
- 需要导航事件追踪

### 实施步骤

1. **创建导航钩子**

```tsx
// hooks/useNavigation.ts
'use client'

import { useRouter, usePathname } from 'next/navigation'
import { useCallback, useState } from 'react'

export function useNavigation() {
  const router = useRouter()
  const pathname = usePathname()
  const [isNavigating, setIsNavigating] = useState(false)

  const navigate = useCallback((href: string) => {
    // 相同页面不导航
    if (href === pathname) return

    setIsNavigating(true)

    try {
      // 尝试客户端导航
      router.push(href)
      
      // 设置超时检测
      const timeoutId = setTimeout(() => {
        // 如果3秒后还在同一页面，说明导航失败
        if (window.location.pathname === pathname) {
          console.warn('客户端导航失败，降级到浏览器导航')
          window.location.href = href
        }
        setIsNavigating(false)
      }, 3000)

      // 监听路由变化
      const checkInterval = setInterval(() => {
        if (window.location.pathname !== pathname) {
          clearTimeout(timeoutId)
          clearInterval(checkInterval)
          setIsNavigating(false)
        }
      }, 100)

    } catch (error) {
      console.error('导航错误:', error)
      // 直接降级到浏览器导航
      window.location.href = href
      setIsNavigating(false)
    }
  }, [pathname, router])

  return { navigate, isNavigating }
}
```

2. **创建智能 Link 组件**

```tsx
// components/SmartLink.tsx
'use client'

import { useNavigation } from '@/hooks/useNavigation'

interface SmartLinkProps {
  href: string
  children: React.ReactNode
  className?: string
  prefetch?: boolean
}

export default function SmartLink({ 
  href, 
  children, 
  className,
  prefetch = false
}: SmartLinkProps) {
  const { navigate, isNavigating } = useNavigation()

  const handleClick = (e: React.MouseEvent<HTMLAnchorElement>) => {
    e.preventDefault()
    navigate(href)
  }

  return (
    <a 
      href={href} 
      onClick={handleClick}
      className={`${className} ${isNavigating ? 'opacity-50 cursor-wait' : ''}`}
      data-prefetch={prefetch}
    >
      {children}
    </a>
  )
}
```

3. **添加导航状态指示器**

```tsx
// components/NavigationIndicator.tsx
'use client'

import { useNavigation } from '@/hooks/useNavigation'

export default function NavigationIndicator() {
  const { isNavigating } = useNavigation()

  if (!isNavigating) return null

  return (
    <div className="fixed top-0 left-0 right-0 h-1 bg-blue-500 z-50">
      <div className="h-full bg-blue-300 animate-pulse" />
    </div>
  )
}
```

### 优点
- 智能降级机制
- 保留客户端导航的可能性
- 提供导航状态反馈

### 缺点
- 实现相对复杂
- 可能有延迟（等待超时）
- 需要更多的客户端代码

## 方案C：优化构建配置

### 适用场景
- 可以控制构建和部署流程
- 希望保留 Next.js 的所有功能
- 有技术能力调试构建问题

### 实施步骤

1. **更新 next.config.js**

```js
/** @type {import('next').NextConfig} */
const nextConfig = {
  output: 'export',
  
  // 优化静态导出
  experimental: {
    // 启用增量静态再生成的实验性支持
    incrementalCacheHandlerPath: false,
  },
  
  // 确保正确的基础路径
  basePath: process.env.BASE_PATH || '',
  
  // 资源前缀
  assetPrefix: process.env.ASSET_PREFIX || '',
  
  // 生成构建ID
  generateBuildId: async () => {
    return process.env.BUILD_ID || `build-${Date.now()}`
  },
  
  // 禁用图片优化（静态导出不支持）
  images: {
    unoptimized: true,
  },
  
  // 追踪模式
  trailingSlash: true,
  
  // 页面扩展名
  pageExtensions: ['tsx', 'ts', 'jsx', 'js'],
  
  // Webpack 配置
  webpack: (config, { buildId, dev, isServer, defaultLoaders, webpack }) => {
    // 添加别名
    config.resolve.alias = {
      ...config.resolve.alias,
      '@': __dirname,
    }
    
    // 优化块分割
    if (!isServer) {
      config.optimization.splitChunks = {
        chunks: 'all',
        cacheGroups: {
          default: {
            minChunks: 2,
            priority: 10,
            reuseExistingChunk: true,
          },
          commons: {
            test: /[\\/]node_modules[\\/]/,
            name: 'vendor',
            priority: 20,
          },
        },
      }
    }
    
    return config
  },
}

module.exports = nextConfig
```

2. **创建构建脚本**

```bash
#!/bin/bash
# scripts/build-static.sh

echo "🚀 开始构建静态站点..."
```

3. **添加后处理脚本**

```js
// scripts/post-build.js
const fs = require('fs')
const path = require('path')

function processHtmlFiles(dir) {
  const files = fs.readdirSync(dir)
  
  files.forEach(file => {
    const filePath = path.join(dir, file)
    const stat = fs.statSync(filePath)
    
    if (stat.isDirectory()) {
      processHtmlFiles(filePath)
    } else if (file.endsWith('.html')) {
      let content = fs.readFileSync(filePath, 'utf8')
      
      // 修复相对路径
      content = content.replace(/href="\//g, 'href="./')
      content = content.replace(/src="\//g, 'src="./')
      
      // 添加基础标签
      if (!content.includes('<base')) {
        content = content.replace(
          '<head>',
          '<head>\n  <base href="/" />'
        )
      }
      
      fs.writeFileSync(filePath, content)
    }
  })
}

console.log('🔧 执行构建后处理...')
processHtmlFiles(path.join(__dirname, '../out'))
console.log('✅ 后处理完成！')
```

- 保留 Next.js 的优化功能
- 更好的性能和用户体验
- 适合长期维护

### 缺点
- 配置复杂
- 需要深入理解 Next.js
- 可能需要多次调试

## 方案D：服务器端配置

### 适用场景
- 使用 Nginx、Apache 等 Web 服务器
- 有服务器配置权限
- 需要更复杂的路由规则

### 实施步骤

1. **Nginx 配置**

```nginx
server {
    listen 80;
    server_name your-domain.com;
    root /var/www/your-blog/out;
    
    # 启用 gzip
    gzip on;
    gzip_types text/plain text/css text/xml text/javascript 
               application/javascript application/xml+rss 
               application/json;
    
    # 缓存静态资源
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
    
    # 处理 Next.js 路由
    location / {
        try_files $uri $uri.html $uri/index.html /index.html;
    }
    
    # 处理 API 路由（如果有）
    location /api {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
    
    # 安全头
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
}
```

2. **Apache 配置 (.htaccess)**

```apache
<IfModule mod_rewrite.c>
  RewriteEngine On
  
  # 强制 HTTPS
  RewriteCond %{HTTPS} off
  RewriteRule ^(.*)$ https://%{HTTP_HOST}/$1 [R=301,L]
  
  # 处理目录
  RewriteCond %{REQUEST_FILENAME} -d
  RewriteRule ^(.+[^/])$ /$1/ [R=301,L]
  
  # 处理 HTML 文件
  RewriteCond %{REQUEST_FILENAME} !-f
  RewriteCond %{REQUEST_FILENAME} !-d
  RewriteCond %{REQUEST_FILENAME}.html -f
  RewriteRule ^(.+)$ $1.html [L]
  
  # 处理 index.html
  RewriteCond %{REQUEST_FILENAME} !-f
  RewriteCond %{REQUEST_FILENAME} !-d
  RewriteCond %{REQUEST_FILENAME}/index.html -f
  RewriteRule ^(.+)$ $1/index.html [L]
  
  # 回退到主页
  RewriteCond %{REQUEST_FILENAME} !-f
  RewriteCond %{REQUEST_FILENAME} !-d
  RewriteRule . /index.html [L]
</IfModule>

# 启用压缩
<IfModule mod_deflate.c>
  AddOutputFilterByType DEFLATE text/html text/plain text/xml text/css text/javascript application/javascript application/json
</IfModule>

# 缓存控制
<IfModule mod_expires.c>
  ExpiresActive On
  ExpiresByType image/jpg "access plus 1 year"
  ExpiresByType image/jpeg "access plus 1 year"
  ExpiresByType image/gif "access plus 1 year"
  ExpiresByType image/png "access plus 1 year"
  ExpiresByType image/svg+xml "access plus 1 year"
  ExpiresByType text/css "access plus 1 month"
  ExpiresByType application/javascript "access plus 1 month"
</IfModule>
```

## 故障排除指南

### 问题1：链接点击无响应

**症状**：
- 点击链接后页面不跳转
- 控制台无错误信息
- URL可能会短暂改变后恢复

**解决方案**：
1. 检查浏览器控制台是否有 JavaScript 错误
2. 验证 `out` 目录中是否存在目标页面文件
3. 使用方案A强制浏览器导航作为临时解决方案
4. 检查是否有冲突的事件监听器

### 问题2：404错误

**症状**：
- 控制台显示 404 错误
- 页面资源加载失败
- 可能看到 `/_next/static/` 路径的 404

**解决方案**：
1. 确认 `basePath` 和 `assetPrefix` 配置正确
2. 检查部署平台的路由配置
3. 验证构建输出的文件结构
4. 使用绝对路径或添加 base 标签

### 问题3：样式丢失

**症状**：
- 页面可以访问但样式错乱
- CSS 文件 404
- 字体或图片无法加载

**解决方案**：
1. 检查 CSS 文件路径是否正确
2. 确认静态资源被正确复制到 `out` 目录
3. 验证 `public` 目录的内容是否被包含
4. 检查 CSS 中的相对路径引用

### 问题4：状态丢失

**症状**：
- 页面跳转后状态重置
- 表单数据丢失
- 用户登录状态丢失

**解决方案**：
1. 使用 localStorage 或 sessionStorage 持久化状态
2. 实现状态恢复机制
3. 考虑使用方案B保留部分客户端导航
4. 对关键数据使用 URL 参数传递
