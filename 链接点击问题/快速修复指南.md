# Next.js 15 链接问题快速修复指南

## 🚨 问题描述

你的 Next.js 15 静态导出站点中，点击链接没有反应？需要手动刷新才能看到新页面？

**你遇到的是 Next.js 静态导出的常见问题。** 本指南提供 5 分钟内可以实施的解决方案。

## ⚡ 最快解决方案（2分钟）

### 步骤 1：创建自定义链接组件

创建文件 `components/CustomLink.tsx`：

```tsx
'use client'

export default function CustomLink({ href, children, className }: any) {
  return (
    <a 
      href={href} 
      onClick={(e) => {
        if (!href.startsWith('http')) {
          e.preventDefault()
          window.location.href = href
        }
      }}
      className={className}
    >
      {children}
    </a>
  )
}
```

### 步骤 2：快速替换所有 Link 组件

在项目根目录运行：

```bash
# 1. 替换导入语句
find app -name "*.tsx" -o -name "*.jsx" | xargs sed -i '' "s/import Link from 'next\/link'/import CustomLink from '@\/components\/CustomLink'/g"

# 2. 替换组件标签
find app -name "*.tsx" -o -name "*.jsx" | xargs sed -i '' "s/<Link/<CustomLink/g"
find app -name "*.tsx" -o -name "*.jsx" | xargs sed -i '' "s/<\/Link>/<\/CustomLink>/g"
```

### 步骤 3：重新构建并测试

```bash
npm run build
npm run start
```

**完成！** 你的链接现在应该可以正常工作了。

## 🎯 手动替换方法（如果自动替换失败）

### 1. 找到所有使用 Link 的文件

```bash
grep -r "from 'next/link'" app/
```

### 2. 手动替换每个文件

将：
```tsx
import Link from 'next/link'

<Link href="/posts/example">
  点击这里
</Link>
```

改为：
```tsx
import CustomLink from '@/components/CustomLink'

<CustomLink href="/posts/example">
  点击这里
</CustomLink>
```

## 🔧 常见问题快速解答

### Q: 为什么会出现这个问题？

**A:** Next.js 15 的静态导出（`output: 'export'`）不包含客户端路由所需的运行时代码。Link 组件依赖这些代码进行页面切换。

### Q: 这个修复会影响性能吗？

**A:** 是的，会失去客户端导航的性能优势（无刷新切换）。但对于静态博客来说，影响很小。

### Q: 有更好的解决方案吗？

**A:** 有，但需要更多配置。这个方案是最快最简单的。如需更优方案，请查看[综合部署指南](./综合部署指南.md)。

## 🚀 部署建议

### Vercel 部署

1. 推送代码到 GitHub
2. 在 Vercel 导入项目
3. 不需要特殊配置，直接部署

### Netlify 部署

1. 构建命令：`npm run build`
2. 发布目录：`out`
3. 添加 `_redirects` 文件：
   ```
   /* /index.html 200
   ```

### GitHub Pages 部署

1. 安装 gh-pages：
   ```bash
   npm install --save-dev gh-pages
   ```

2. 添加部署脚本到 `package.json`：
   ```json
   {
     "scripts": {
       "deploy": "npm run build && gh-pages -d out"
     }
   }
   ```

3. 运行部署：
   ```bash
   npm run deploy
   ```

## 💡 优化建议

### 1. 添加加载状态（可选）

```tsx
'use client'
import { useState } from 'react'

export default function CustomLink({ href, children, className }: any) {
  const [isLoading, setIsLoading] = useState(false)
  
  return (
    <a 
      href={href} 
      onClick={(e) => {
        if (!href.startsWith('http')) {
          e.preventDefault()
          setIsLoading(true)
          window.location.href = href
        }
      }}
      className={`${className} ${isLoading ? 'opacity-50 cursor-wait' : ''}`}
    >
      {children}
    </a>
  )
}
```

### 2. 保留滚动位置（可选）

```tsx
onClick={(e) => {
  if (!href.startsWith('http')) {
    e.preventDefault()
    // 保存滚动位置
    sessionStorage.setItem('scrollPos', window.scrollY.toString())
    window.location.href = href
  }
}}
```
